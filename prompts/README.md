# Prompts

Reusable instructions that bias generation toward expert technique.

A prompt here is a self-contained block of text you can paste into (or reference from) an AI coding
session to push output toward the upper tail of quality, rather than the median.

## What belongs here

- Standalone prompts that improve a class of output (e.g. error handling, API design, test design).
- Prompt fragments meant to be composed into larger instructions.

## Conventions

- One prompt per file; name it for what it does (`expert-error-handling.md`).
- Lead with a short rationale: *why* this prompt pulls from the upper tail.
- Include a concrete before/after example where practical.
- Record the model, tool, and version you validated it against — results are version-specific.
