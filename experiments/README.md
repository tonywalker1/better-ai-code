# Experiments

Reproducible evidence on what actually moves output quality — and what doesn't.

This is the project's history and proving ground. An artifact (prompt, skill, or rule) earns its
place by demonstrating an effect here. Negative results are first-class: an artifact that fails to
help is a finding worth recording.

## What belongs here

- Before/after comparisons isolating the effect of a single artifact.
- Notes on what was tried, what was observed, and the conclusion drawn.

## Conventions

- One experiment per subdirectory, dated and named for the question (`2026-06-23-error-handling/`).
- Record the hypothesis, the exact model/tool/version, the prompt(s) used, and the inputs.
- Capture concrete output (before and after) so the result is reproducible and reviewable.
- State the conclusion plainly, including null and negative results.
